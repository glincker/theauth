KavachOS is a TypeScript SDK that gives AI agents their own identity, enforces permissions at call time, and logs every decision. It plugs into whatever human auth you already run (Clerk, Auth.js, better-auth) and takes over once the human is signed in.

```ts
import { createKavach } from 'kavachos';

const kavach = await createKavach({
  database: { provider: 'sqlite', url: 'kavach.db' },
});

const agent = await kavach.agent.create({
  ownerId: user.id,
  name: 'code-reviewer',
  type: 'autonomous',
  permissions: [{ resource: 'mcp:github:*', actions: ['read'] }],
});

const result = await kavach.authorize(agent.id, {
  action: 'read',
  resource: 'mcp:github:repos',
});
// { allowed: true, auditId: '550e8400-...' }
```

## How it fits

Your auth provider handles human login and returns a user ID. That user ID is what KavachOS hangs agents off of. When an agent wants to act, your code calls `authorize()` with the agent ID, the action, and the resource. KavachOS returns `{ allowed, reason, auditId }` and writes a row to the audit log either way.

```mermaid
flowchart LR
  User([Human user]) -->|signs in| HumanAuth[Clerk / Auth.js / better-auth]
  HumanAuth -->|user ID| Kavach[KavachOS]
  Kavach -->|agent.create| Agents[(Agents)]
  Kavach -->|authorize| Decision{allowed?}
  Decision -->|yes| Tools[MCP servers, APIs, databases]
  Decision -.->|logged either way| Audit[(Audit trail)]
```

```ts
// After Clerk / Auth.js / better-auth verifies the human:
const agent = await kavach.agent.create({ ownerId: user.id });

// Later, when the agent calls a tool:
const { allowed, reason, auditId } = await kavach.authorize(agent.id, {
  action: 'read',
  resource: 'mcp:github:repos',
});
```

<Info>
KavachOS does not replace your human auth. It does not handle login forms, sessions, or social OAuth. It starts where human auth ends.
</Info>
