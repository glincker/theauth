# SOC 2 and ISO 42001

## SOC 2 Type II

SOC 2 Type II audits evaluate the operating effectiveness of controls over a period of time. TheAuth provides the technical infrastructure for several relevant trust service criteria.

### Relevant criteria

| Criteria | Description | TheAuth feature |
|---|---|---|
| CC6.1 | Logical and physical access controls | Bearer tokens with SHA-256 hash storage; no plaintext recovery after issuance |
| CC6.3 | Access revocation | Revocation is immediate and permanent; takes effect on the next `authorize()` call |
| CC7.2 | Anomalies and events monitored | Anomaly detection on high-frequency and high-denial-rate patterns; off-hours access flagging |
| CC9.2 | Risk monitoring and evaluation | Trust scoring per agent; five-level classification from audit log |

### Audit evidence

The `kavach_audit_logs` table is append-only and records every authorization decision. Export it with `kavach.audit.export({ format: 'csv' })` to produce evidence for auditors.

For a SOC 2 audit covering AI agent operations, the audit log satisfies the event logging requirement under CC7.2.

## ISO 42001

ISO 42001 (AI Management System) is the first international standard specifically for AI management systems, published in December 2023.

### Relevant controls

| Control area | ISO 42001 requirement | TheAuth feature |
|---|---|---|
| Access to AI systems | Documented access controls | Permission engine with resource/action model |
| Monitoring and measurement | Audit logging of AI system actions | `kavach_audit_logs`, append-only |
| Risk treatment | Controls to address identified risks | Permission constraints (rate, IP, time window, approval) |
| Human oversight | Mechanisms for human intervention | `requireApproval` constraint, delegation depth limits |

## Evidence collection

### Exporting for auditors

```typescript
// Full audit log export
const csv = await kavach.audit.export({ format: 'csv' });

// Specific time range for the audit period
const periodExport = await kavach.audit.export({
  format: 'csv',
  since: new Date('2026-01-01'),
  until: new Date('2026-06-30'),
});
```

### Demonstrating access control

```typescript
// Show what permissions an agent holds
const agent = await kavach.agent.get(agentId);
console.log(agent.permissions);

// Show effective permissions including delegations
const effective = await kavach.delegation.getEffectivePermissions(agentId);
console.log(effective);
```

!!! warning "Verify"
    SOC 2 and ISO 42001 audits cover your entire system, not just the auth layer. Consult your auditor about which controls TheAuth evidence satisfies versus controls you must implement in your own application code.
